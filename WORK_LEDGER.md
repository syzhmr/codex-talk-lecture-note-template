# 作業台帳

このファイルは，資料台帳，講義回対応，ページ別監査，未解決点，参考文献対応，記号修正をまとめる。
本文方針は `NOTE_POLICY.md`，作業入口は `WORKFLOW_STEPS.md`，参考文献 PDF の棚卸しは `references/REFERENCE_INDEX.md` を参照する。

## 運用状態

`NOTE_POLICY.md` のスモールステップを実作業に使う。
作業を始めるモデルは，まず「作業キュー」と「スモールステップ進捗」を見て，対象ブロックと次の Step を固定する。
作業を止める場合は，本文差分の有無にかかわらず，次のモデルが再開できるように「次の Step」と停止理由を残す。

| 状態 | 意味 |
| --- | --- |
| 未着手 | まだ Step 1 に入っていない。 |
| 進行中 | 対象を固定し，途中の Step まで進んでいる。 |
| 保留 | 判読不能，資料不足，ユーザー判断待ちなどで止めている。 |
| 本文整理済み | TeX 反映と最低限の検査まで終わっている。完成扱いとは限らない。 |
| 完成 | ユーザー確認または明示的な完成判定を得ている。 |
| ロック済み | 通常改訂では変更しない。明示指示がある場合だけ触る。 |

## 作業ログテンプレート

作業ごとに必要な場合だけ，この形式で短いログを追加する。
長い本文案はここに貼らず，抽出結果，判断，次の Step を残す。

```md
### YYYY-MM-DD / [単位] / Step [番号]

- 対象:
- 確認資料:
- 抽出した内容:
- 参考文献・記号対応:
- 目標密度・loss audit:
- TeX 反映:
- 検査:
- 未解決点:
- 次の Step:
```
