単発講演にも転用できるが，標準運用では集中講義の日程・講義回の流れを優先する。

## 状態の目安

| 状態 | 意味 |
| --- | --- |
| 未確認 | まだ資料を個別確認していない。 |
| 資料確認中 | 手書きノートは見たが，板書・スライド・配布資料との照合が未完了。 |
| 参考候補抽出済み | 手書きノート由来の参考文献候補を `HANDWRITTEN_REFERENCE_INDEX.md` に記録済み。 |
| 主張対応中 | 講義中の主張と参考文献の対応を確認中。 |
| 本文整理済み | 本文へ整理し，反映先を記録済み。 |
| 要再確認 | 判読不能，資料差し替え，資料間衝突，引用未特定がある。 |
| 対応なし | その範囲に該当資料がない。 |

## 運用

- 作業単位は話題ブロックにする。
- 集中講義では，まず日程と講義回を固定し，その中で話題ブロックを設定する。
- 話題ブロックは，講義中の流れ，手書きノートの区切り，スライドの節，板書の切れ目を見て設定する。
- 話題ブロックを整理するときも，講義回をまたいで教科書風に再編成しない。
- 広い範囲を一括整理しない。
- 本文整理前に，手書きノート由来の参考文献候補と講義中の主張候補を確認する。
- ブロックをまたぐ記号衝突は `NOTATION_CONFLICTS.md` に記録する。
