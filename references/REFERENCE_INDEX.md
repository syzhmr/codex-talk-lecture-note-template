## 棚卸しルール

- 「存在と目次を確認した」ことと「講義中の主張に対応する箇所を確認した」ことを区別する。
- ユーザーが追加した PDF，URL，紙媒体情報は，未確認のまま放置せず，まず所在，分野・概念，確認状況を登録する。
- 参考文献 PDF は，プロジェクト同期に必要な資料として原則 Git で追跡する。
- GitHub の 100MB 制限，公開範囲，権利上の問題がある資料だけを個別例外として除外し，理由，代替ファイル，分割方針，保管場所を記録する。
- 講義中の主張に対応させる文献は，定理・命題の仮定，結論，番号，ページを確認する。
- 証明に使うためではなく，引用箇所の特定と対応関係の確認のために精読する。
- 手書きノートと配布資料に入っている参考文献を主引用候補として最優先にする。
- 一般的に飛ばされている背景や標準事実を確認する文献は，補足引用として分類する。
- 手書きノートの色付き矢印や略記から文献候補を拾った場合は，`WORK_LEDGER.md` に記録する。
- Codex が提案した参考文献候補は，`WORK_LEDGER.md` に記録し，講義由来の文献候補やユーザー追加資料と混ぜない。
- 参考文献欄には URL または DOI を可能な限り明示し，資料紹介文や用途説明はこの台帳に分離する。
- 参考文献候補が判読できない場合は，本文に曖昧に混ぜず，`WORK_LEDGER.md` に残す。
- 使用していない文献を削除候補にする場合は，「本文未反映」と「削除候補」を区別して理由を書く。
