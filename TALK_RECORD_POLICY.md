# 講演記録作成方針

このファイルは，集中講義で記録された手書きノートを，後から読める集中講義記録として TeX 化するための全体方針である。
作業の入口は必ず `WORKFLOW_STEPS.md` とし，プロジェクト固有情報は `PROJECT_PROFILE.md`，資料の所在は `SOURCE_INDEX.md`，日程・講義回・話題ブロックの対応は `SESSION_INDEX.md` を優先する。
単発講演にも転用できるが，運用上の判断は集中講義記録を主対象として行う。

## 基本方針

- 完成版は，手書きノートの PDF 化を主眼にする。
- 手書きノートに書かれている内容は，判読不能箇所を除き，すべて TeX 打ちする。
- 日程，講義回，節題，話題ブロックの順序を保存し，教科書風の章構成へ勝手に組み替えない。
- 節題，小見出し，番号付けは手書きノートに従う。
- 講義中に記録された流れ，主張，記号，図式，引用先，参考文献対応を重視する。
- 用語と表現は手書きノートを優先し，手書きノートで英語表記の語は無理に日本語へ訳さない。
- 手書きノートを最優先資料にする。
- 板書写真，スライド，配布資料は，手書きノートの記録を確認・補強するために使う。
- ユーザーが `sources/` や `references/` に入れた資料は，台帳化した上で，対象範囲に関係する限り積極的に使う。
- 参考文献は，講義中の主張がどの文献のどの定理，命題，節，ページ，URL に対応するかを確認するために使う。
- 手書きノートと配布資料に入っている参考文献を最優先し，講義由来の引用候補として確認する。
- Codex が講義資料の外から提案した参考文献は，講義由来の文献と区別し，引用未特定，記号衝突，標準出典確認，書誌情報確認の補助として使う。
- 一般的に飛ばされている背景や行間を確認する文献は，補足引用に留め，講義記録の主線を置き換えない。
- 証明の不足部分を Codex が補完しない。
- 参考文献を見て，講義中に記録されていない中間補題，証明手順，背景定理を本文へ勝手に追加しない。
- 講演者が述べたと確定できない内容を，講演者の主張として書かない。
- 記号，式，定義の順序は，原則として講義中の記録を基準にする。
- 参考文献側の記法に合わせて講義記録の記法を不用意に置き換えない。
- 記号衝突がある場合は `NOTATION_CONFLICTS.md` に記録する。

## 完成版の性格

完成版は録音の逐語録ではない。
ただし，手書きノートに書かれた内容については，本文へ TeX 打ちすることを基本とする。
講義中に記録されていない内容を滑らかに補って教科書風にすることは目的ではない。

完成版では次を目指す。

- 集中講義の日程，講義回，節題，話題順が追える。
- 手書きノートの記述が，抜粋ではなく本文として残っている。
- 講義中に強調された主張，例，注意，記号が残っている。
- 参考文献の対応箇所が分かる。
- 未確認箇所や資料間衝突を本文に曖昧に混ぜない。
- 本文だけで最低限読めるが，証明の不足は不足として扱う。

## 本文に入れるもの

- 講義中に記録された定義，主張，命題，補題，定理，例，注意。
- 手書きノートに書かれた式，計算，証明スケッチ，欄外メモ，矢印，図式。
- 手書きノートの節題，小見出し，番号付け。
- 手書きノートに残っている講義の流れ。
- 板書写真で確認できる式，図式，見出し，記号。
- スライドや配布資料で確認できる正式な主張，図，参考文献。
- 講義中に参照された文献の対応箇所。
- 参考文献で確認した記号差分，仮定差分，定理番号。
- 一般的に飛ばされている背景を確認するための最小限の補足引用。

本文に補足を入れる場合は，手書きノート本文と補足を区別する。
補足は，証明や紹介された主張・事実が参考文献のどこにあるかを示すためのものとし，手書きノート本文を置き換えない。

## 本文に入れないもの

- 講義中に記録されていない証明補完。
- 参考文献だけから得た追加の中間補題。
- Codex 提案参考文献だけから得た講義外の補足。
- 補足引用を根拠にした本文の大幅な拡張。
- 講義の流れにない背景説明。
- 手書きノートの節題や用語を，PDF の構成に合わせて言い換えた見出し。
- 講演者が述べたと確定できない解釈。
- 仮定や結論の一致を確認していない参考文献由来の主張。
- 判読できない URL，文献名，定理番号を推測で復元したもの。

## 参考文献対応

講義中の主張を本文へ整理する前に，必要に応じて `CLAIM_REFERENCE_MAP.md` を更新する。
対応文献が見つかった場合でも，次を区別する。

- 完全一致
- 記号違いだが内容一致
- 仮定が一部違う
- 結論が類似
- 関連箇所だが対応未確定
- 未発見

本文では，対応が確定したものだけを明確な引用として使う。
未確定のものは本文に混ぜず，台帳に残す。
補足引用は，講義中の主張に直接対応する主引用と区別し，必要最小限の注または短い確認に留める。

## 資料名の扱い

PDF 本文では「手書きノートでは」「スライドでは」「板書では」と作業資料名を不用意に出さない。
ただし，集中講義記録として資料差分や出典の性質を示す必要がある場合は，脚注または作業台帳に分離する。

作業台帳では，資料名，ページ，画像番号，スライド番号，配布資料の節番号を明確に書く。

## 用語・記号

- 講義中の記法を最優先にする。
- 用語は手書きノートの表現を最優先にする。
- 手書きノートで英語表記の用語は，必要がない限り英語のまま本文に残す。
- 日本語化する場合は，手書きノート内に日本語表記がある場合を優先し，なければ `TERM_DICTIONARY_CANDIDATES.md` に候補として残す。
- 手書きノートの記号が不明確な場合は，板書，スライド，配布資料で確認する。
- 参考文献の記号と講義中の記号が異なる場合は，どちらを採用したかを `NOTATION_CONFLICTS.md` に記録する。
- 用語が未確定または不自然な場合は，本文へ直接入れず `TERM_DICTIONARY_CANDIDATES.md` に回す。

## 査読

通常査読では，手書きノート，板書写真，スライド，配布資料，参考文献を見てよい。
独立査読では，完成版 PDF，引用箇所，参考文献対応，記号衝突を中心に確認する。
独立査読で出た修正文案，判断が分かれる補足，資料間の衝突は `REVISION_CANDIDATES.md` または `SOURCE_GAPS.md` に記録する。

## 報告

作業後は，次を簡潔に報告する。

- 対象日程・講義回・話題ブロック。
- 使用した手書きノート，板書，スライド，配布資料。
- 抽出した参考文献候補。
- 対応付けた講義中の主張と参考文献箇所。
- 未確定の引用，記号衝突，資料間衝突。
- 本文変更とビルド結果。
