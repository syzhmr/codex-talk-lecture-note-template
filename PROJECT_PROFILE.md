# プロジェクト設定

このファイルは，テンプレートをコピーした後に最初に埋めるプロジェクト固有の設定である。
Codex は `WORKFLOW_STEPS.md` を入口として読み，このファイルで対象講演・集中講義の前提を確認する。

## 本文中の日付表示

- 目次には日付を反映しない。
- 各日の最初のセクション見出しの直前に `\recorddate{YYYY年M月D日}` の形で日付を表示する。
- 日付は記録性を高めるための本文情報として扱い，節題や小見出しとは分け，太字にはしない。
- 概要と講義本文はひとつなぎにし，概要後に強制改ページを置かない。日付は直後の節見出しに付くように配置する。

## 資料優先順位

このプロジェクトでは，原則として次の順に資料を確認する。
プロジェクト固有の理由で変更する場合は，ここに理由を記録する。
ユーザーが `sources/` または `references/` に入れた資料は，台帳に登録した上で，対象範囲に関係する限り積極的に確認する。
集中講義記録では，日程，講義回，話題ブロックの順序を本文構成の基準にする。

1. 手書きノート
2. 板書写真
3. スライド
4. 配布資料
5. 講義内で参照された参考文献
6. 既存の参考文献台帳
7. Codex 提案参考文献
8. 必要最小限の外部検索

## 手書きノート

- 配置場所: `sources/handwritten_notes/[ファイル名]`
- 役割: 講義中にユーザーが重要と見た情報，話題の順序，主張，記号，参考文献候補，URL，疑問点を拾うための最優先資料。
- OCR: 手書きノートは OCR にかけない。本文確定は単ページ画像または元 PDF の該当ページで行う。
- 色・記号の意味: [例: 赤字=重要，青字=補足，黄色=参考文献候補，矢印=対応関係]
- 旧版資料の扱い: [旧版ノートや古い抽出画像を使うか，使わないか]

## 板書写真

- 配置場所: `sources/board_photos/[ファイル名]`
- 役割: 講義中の式，図式，記号，証明の見出し，定義，主張の順序を確認する補助資料。
- 注意: 板書だけを根拠に，手書きノートにない証明補完を行わない。

## スライド

- 配置場所: `sources/slides/[ファイル名]`
- 役割: 講演者が用意した定理名，主張，図，参考文献，URL，公式な用語を確認する補助資料。
- 注意: スライドにない講義中発言を推測で補わない。

## 配布資料

- 配置場所: `sources/handouts/[ファイル名]`
- 役割: 講義の正式な補助資料として，定義，定理番号，仮定，参考文献を確認する。

## 参考文献

- 参考文献 PDF は `references/pdfs/` に置く。
- 参考文献候補は，まず `WORK_LEDGER.md` に記録する。
- Codex が講義資料の外から提案した参考文献候補も，講義由来候補と出自を分けて `WORK_LEDGER.md` に記録する。
- 参考文献の棚卸しは `references/REFERENCE_INDEX.md` に書く。
- 講義中の主張と参考文献の対応は `WORK_LEDGER.md` に書く。
- 本文で引用する場合は，節番号，定理番号，命題番号，ページ，URL を確認する。
- 未確認のページ番号・定理番号を推測で書かない。
- 手書きノートと配布資料に入っている参考文献を最優先の引用候補にする。
- 一般的に飛ばされている背景や行間を確認する文献は，補足引用に留める。

## 本文方針

- 手書きノートの内容をすべて TeX 打ちする。
- 節題，小見出し，番号付けは手書きノートに従う。
- 完成版は集中講義記録寄りにする。
- 講義で記録された流れ，主張，記号，引用先を重視する。
- 可換図式では，通常の同型関係に `\cong`，射上の同型ラベルに `\sim` を用い，縦矢印の同型ラベルは矢印方向に合わせて回転させる。元や所属記号が書かれた図式では，元の位置・`\in` の向き・矢印ラベル・同型記号を図式の一部として扱い，本文の文章だけで省略しない。
- チルダ系の `\tilde{}`，`\widetilde{}`，`\sim` とハット系の `\hat{}`，`\widehat{}` は必ず区別して照合する。
- 手書きノートの黄色・赤・青・黒などの矢印，引き出し線，丸囲み，下線は，数学的補足，注意，参考文献候補，修正指示のいずれかとして必ず確認し，本文または台帳へ反映する。
- 証明の不足部分を Codex が補完しない。
- 数学的に自然な補いであっても，手書きノートにない主張，理由，証明範囲，接続文，抽象化表現を本文へ足さない。
- 手書きノートの日本語が判読できる場合は，その表現を優先し，日本語参考文献は判読不能箇所や未確定候補の表現確認に限って使う。
- `レポート問題`，`これを示せ`，欄外メモ，矢印，引き出し線は，画像上の位置から作用範囲を確認し，不明な場合は本文で広げず台帳に残す。
- 参考文献は引用箇所の特定と記号衝突の確認に使う。
- 参考文献欄には URL または DOI を可能な限り明示する。
- 集中講義中に挙げられた参考文献とユーザー追加参考文献を分ける。
- Codex 提案参考文献は，講義由来の文献候補を置き換えず，引用未特定，記号衝突，標準出典確認，書誌情報確認の補助として扱う。
- 本文に「手書きノートでは」「スライドでは」と作業資料名を不用意に出さない。
- ただし，引用箇所の明示や資料差分の説明が必要な場合は，脚注または作業台帳に分離する。

- 記号衝突: `WORK_LEDGER.md`

## 作業単位

- 集中講義では講義回内の話題ブロック単位で整理する。
- 単発講演では話題ブロック単位で整理する。
- 広い範囲の依頼がある場合でも，手書きノート，板書，スライド，配布資料，参考文献対応を確認できる範囲に分割する。
