# 参考文献引用方針

このファイルは，講義中に参照された文献，手書きノートに書かれた文献候補，講義中の主張と参考文献の対応を管理するための方針である。

## 基本方針

- 最初に手書きノートから参考文献候補を拾う。
- 手書きノートと配布資料に入っている参考文献を最優先の引用候補として扱う。
- スライド，配布資料，板書写真にある参考文献欄，脚注，URL と照合する。
- 講義中に示された主張が，どの参考文献のどの定理，命題，補題，節，ページ，URL に対応するかを確認する。
- 対応箇所は `CLAIM_REFERENCE_MAP.md` に記録する。
- 参考文献は証明補完のためではなく，引用箇所の特定，主張の対応確認，記号衝突の確認のために使う。
- 一般的に飛ばされている背景や行間を確認するための文献は，本文の主線を拡張せず，補足程度の引用に留める。
- Codex が講義資料の外から参考文献を提案する場合は，講義由来の参考文献候補と出自を分けて `CODEX_REFERENCE_CANDIDATES.md` に記録する。
- 定理番号，命題番号，ページ番号，URL は推測で書かない。

## 探す順序

1. 手書きノート内の文献名，URL，略記，定理番号。
2. 配布資料の参考文献，注釈。
3. スライド内の参考文献欄，脚注，URL。
4. 板書写真に写っている文献名，定理名，URL。
5. `HANDWRITTEN_REFERENCE_INDEX.md`。
6. `references/REFERENCE_INDEX.md`。
7. `references/notes/README.md`。
8. `CODEX_REFERENCE_CANDIDATES.md`。
9. 必要最小限の外部検索。

外部検索へ進む前に，講義中に実際に参照された痕跡を拾う。

## 引用の優先度

引用候補は，出自に応じて次のように扱う。

| 優先度 | 出自 | 扱い |
| --- | --- | --- |
| 最優先 | 手書きノートに書かれた文献・URL・定理番号 | 講義中に参照された痕跡として最初に確認する。 |
| 最優先 | 配布資料に載っている文献・注・参考文献欄 | 講義の正式な補助資料として最初に確認する。 |
| 高 | スライドに載っている文献・脚注・URL | 講演者が提示した公式情報として確認する。 |
| 高 | 板書写真に写っている文献名・定理名・URL | 講義中に扱われた痕跡として確認する。 |
| 中 | ユーザーが `references/` に入れた文献 | 対象範囲との関係を台帳化して確認する。 |
| 補足 | Codex 提案参考文献・外部検索で見つけた文献 | 講義由来の引用候補を置き換えず，補足引用または確認候補に留める。 |

手書きノートまたは配布資料に対応する参考文献候補がある場合，Codex 提案参考文献や一般検索由来の文献を先に本文引用として採用しない。
講義由来の候補が判読不能または未特定の場合も，それを無視せず `HANDWRITTEN_REFERENCE_INDEX.md` または `SOURCE_GAPS.md` に残す。

## 補足引用

一般的に飛ばされている背景，標準事実，記号の由来，既知の定理名を確認するための文献は，補足引用として扱う。
補足引用は，講義記録の主線を置き換えたり，講義中に記録されていない証明を補完したりするためには使わない。

補足引用を使う場合は，次を守る。

- 手書きノート，配布資料，スライド，板書に対応する講義由来の引用候補を先に確認する。
- 補足引用であることを `CLAIM_REFERENCE_MAP.md` または `CODEX_REFERENCE_CANDIDATES.md` に記録する。
- 本文では必要最小限の注または短い確認に留める。
- 補足引用だけを根拠に，新しい主張，補題，証明手順を本文へ追加しない。
- 講義中の主張と補足引用先の仮定・結論が違う場合は，本文へ混ぜず台帳に残す。

## Codex 提案参考文献

Codex が講義資料の外から参考文献を提案する場合は，`CODEX_REFERENCE_CANDIDATES.md` に記録する。
Codex 提案参考文献は，講義中に実際に参照された文献として扱わない。

Codex 提案参考文献を使ってよい目的は次に限る。

- 手書きノートやスライドから拾った引用候補の URL，書誌情報，定理番号を確認する。
- 講義中の主張に対応する標準的な出典候補を探す。
- 講義中の記号と参考文献側の記号の衝突を確認する。
- 講義中に出た文献略記や不完全な URL の候補を確認する。
- 既存台帳にないが，講義中の引用未特定点を解消する候補を提示する。
- 一般的に飛ばされている背景や標準事実について，補足引用候補を提示する。

Codex 提案参考文献を採用する前に，次を記録する。

- 提案理由。
- 検索語，既存台帳，引用先などの発見経路。
- 候補文献の書誌情報または URL。
- 確認した定理番号，命題番号，節番号，ページ，URL。
- 講義中の主張との関係。
- 本文引用に使うか，台帳だけに残すか。
- 主引用として使うのか，補足引用に留めるのか。

## 手書きノート由来の参考文献候補

手書きノートに次の要素がある場合は，`HANDWRITTEN_REFERENCE_INDEX.md` に記録する。

- 著者名。
- 書名，論文名，講義ノート名。
- URL，DOI，arXiv 番号。
- Stacks tag。
- MathOverflow/Math StackExchange 番号。
- 定理番号，命題番号，補題番号，節番号，ページ番号。
- 文献略記。
- 色付き矢印，引き出し線，丸囲み，下線で示された参考候補。

判読できない候補は，無視せず「未判読参考候補」として残す。

## 主張との対応付け

講義中の主張と参考文献の対応は，`CLAIM_REFERENCE_MAP.md` に記録する。
対応の状態は次のいずれかにする。

- 完全一致
- 記号違いだが内容一致
- 仮定が一部違う
- 結論が類似
- 関連箇所だが対応未確定
- 未発見

完全一致または記号違いだが内容一致とする場合は，仮定，対象，結論が対応していることを確認する。
仮定が異なる場合は，本文では一致として扱わない。

## 本文での引用

本文で参考文献を引用する場合は，可能な限り次を明示する。

- 著者名または文献名。
- 定理，命題，補題，節，ページ。
- URL または DOI。
- 最終確認日。
- 講義中の主張との関係。

講義中の主張と参考文献の対応が未確定の場合，本文へ引用として出さない。
必要なら `SOURCE_GAPS.md` に確認課題として残す。

## 禁止事項

- 参考文献を見て証明を補完しない。
- 講義中に記録されていない補題，定理，証明手順を本文の流れへ勝手に追加しない。
- 対応文献が見つかっただけで，講義中の主張と同一視しない。
- 仮定，記号，結論が違う場合は，本文に混ぜず対応表に記録する。
- 定理番号，ページ番号，URL を推測で書かない。
- 参考文献側の記号を，講義中の記号へ説明なく置き換えない。
- Codex 提案参考文献を，講義中に実際に参照された文献として扱わない。
- Codex 提案参考文献で，手書きノート由来の未判読候補を勝手に置き換えない。

## 記録先

- 参考文献候補: `HANDWRITTEN_REFERENCE_INDEX.md`
- Codex 提案参考文献候補: `CODEX_REFERENCE_CANDIDATES.md`
- 講義中の主張と対応箇所: `CLAIM_REFERENCE_MAP.md`
- 記号衝突: `NOTATION_CONFLICTS.md`
- 判読不能・未確認・資料衝突: `SOURCE_GAPS.md`
- 参考文献の書誌情報: `references/REFERENCE_INDEX.md`
- 参考文献ごとの読解メモ: `references/notes/`
