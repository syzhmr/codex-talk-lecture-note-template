# 講義ノート作成方針

このファイルは，講義・講演記録を本文化するときの品質基準をまとめる。
主資料は手書きノートであり，参考文献は本文を置き換える資料ではなく，講義中の主張の所在，記号差，書誌情報を確認するために使う。

## 入口

1. 作業開始時は `WORKFLOW_STEPS.md` を読む。
2. プロジェクト固有情報は `PROJECT_PROFILE.md` を確認する。
3. 資料，話題ブロック，未解決点，記号修正，参考文献対応は `WORK_LEDGER.md` を確認する。
4. 用語判断は `TERM_DICTIONARY.md` を優先する。
5. LaTeX の構造や整形だけを触る場合は `LATEX_SOURCE_POLICY.md` を確認する。
6. Git 操作は `GIT_WORKFLOW.md` に従う。

## 品質基準

- 完成 PDF は，手書きノートを参照しなくても読める講義ノートにする。
- 日程，講義回，手書きノートの話題順，節題，小見出し，定義・定理・命題などの区切りを保つ。
- 手書きノートにある文章，式，計算，図式，矢印，欄外メモ，証明，証明の sketch は，要約で置き換えない。
- 手書きノートより薄い本文にしない。整理とは短縮ではなく，判読できる内容を読める形で TeX 化することである。
- 手書きノートは OCR にかけない。本文確定は単ページ画像または元 PDF の該当ページを目視して行う。
- 参考文献は，講義中の主張との対応，引用箇所，記号衝突，訳語候補，書誌情報の確認に使う。手書きノートにない証明や背景定理を本文へ補完するためには使わない。
- 本文には「手書きノート p.X」「ノートでは」「記録」「未確認」などの作業用表現を出さない。未解決点は `WORK_LEDGER.md` に分離する。
- 数学的に自然に見える補いでも，手書きノートにない主張，理由，証明範囲，接続文は本文へ足さない。

## 資料優先順位

1. 手書きノート
2. 板書写真
3. スライド
4. 配布資料
5. 講義中に提示された参考文献
6. ユーザー追加参考資料
7. Codex が講義資料の外から提案した補助候補

手書きノートで判読できる表現は，日本語参考文献の標準表現より優先する。
参考文献側の記法，説明順，証明展開は，手書きノートの記法，順序，粒度を置き換える根拠にしない。

## 本文整理

本文整理は，完成済み目次に現れる最小単位を基本単位にする。
目次最小単位が大きい場合だけ，`WORK_LEDGER.md` の話題ブロックまたは手書きページ単位へ内部的に分ける。
広い範囲を依頼された場合でも，複数の最小単位を一度に完成扱いにしない。

各単位では次のスモールステップを飛ばさない。
各ステップの終了時には，出力または停止理由を `WORK_LEDGER.md` の「スモールステップ進捗」または「未解決点」に残す。

| Step | 名前 | やること | 出力 |
| --- | --- | --- | --- |
| 0 | 入口判定 | 依頼型，対象単位，ロック済み範囲，読むファイルを決める。 | 作業型，対象ブロック，読むファイル。 |
| 1 | 対象固定 | 対象ページ，対象 TeX，対応する話題ブロック，関連参考文献，既存未解決点を列挙する。 | 対象ページ，対象資料，編集対象，既存リスク。 |
| 2 | 資料登録確認 | 対象資料が `WORK_LEDGER.md` または `references/REFERENCE_INDEX.md` に登録済みか確認する。 | 登録済み資料と追加登録した資料。 |
| 3 | 手書き抽出 | OCR を使わず，単ページ画像または元 PDF で，見出し，本文，式，図式，欄外注，レポート問題，証明分類を抽出する。 | 本文化する項目，図式候補，判読不能箇所。 |
| 4 | 参考文献候補抽出 | 手書きノートの文献名，URL，定理番号，矢印や下線が指す根拠候補を拾う。 | 参考文献候補，既存文献との対応，未判読候補。 |
| 5 | 主張・記号対応 | 講義中の主張と参考文献の対応，記号差，仮定差を確認する。 | 主張対応，記号修正，本文に入れない補完。 |
| 6 | TeX 反映 | Step 3--5 で確定した内容だけを対象 TeX に反映する。 | TeX 差分。 |
| 7 | 検査 | 作業用表現，未確認ラベル，記号ミス，必要なビルドを確認する。 | 実行した `rg`，ビルド結果，残リスク。 |
| 8 | 台帳更新 | 進捗，未解決点，参考文献対応，記号修正を `WORK_LEDGER.md` に反映する。 | 更新した台帳項目と次の Step。 |
| 9 | 報告 | 対象，確認資料，反映内容，未解決点，検査結果を短く報告する。 | ユーザーへの完了報告。 |

完成扱いにできるのは，手書きノートの対象範囲がページ単位で確認され，本文，参考文献対応，記号差，未解決点の置き場所がすべて決まっている場合だけである。
Step 6 の TeX 反映だけを行っても完成扱いにしない。
Step 8 の台帳更新まで終わっていない場合は，次のモデルが再開できるように「次の Step」を明示する。

## 画像確認とOCR

- 手書きノート，板書写真，手書きメモは OCR にかけない。
- 手書きノートの本文確定，式，図式，矢印，欄外注，レポート問題の範囲判定は，単ページ画像または元 PDF の該当ページを根拠にする。
- 参考文献 PDF の検索には `references/text/` の `*.txt`，`*.ocr.txt`，`*.mmd` を使ってよい。
- 参考文献 PDF の OCR と Nougat は検索補助であり，大きい表示数式，グラフ，可換図式，Hasse 図，引用に関わる符号は原典 PDF の該当ページ画像で確認する。
- OCR や Nougat の結果だけで本文を確定した場合は完成扱いにしない。確認したページ，判断，残リスクを `WORK_LEDGER.md` に残す。

## 図式

- 可換図式は省略しない。
- 可換図式や代数的な矢印図は `tikzcd`，`matrix of math nodes`，直接 `\node` と `\path` による配置の順で TeX 化を検討する。
- グラフ，曲線図，模式図は TikZ を優先する。
- 生成画像では代替しない。
- 元や所属記号がある図式では，元の位置，`\in` の向き，矢印ラベル，同型記号を図式の一部として扱う。
- 射上の同型は `\sim`，通常の同型関係は `\cong` を使う。縦矢印で同型を示す場合は，横向きの `\sim` のまま置かず，矢印方向に合わせて回転させる。

## 参考文献

- 手書きノートに文献名，URL，定理番号，命題番号，節番号，ページ番号がある場合は，まず `WORK_LEDGER.md` の参考文献候補に記録する。
- 講義中の主張と参考文献の対応は，仮定，対象，結論が一致するまで確定扱いにしない。
- 対応は，完全一致，記号違いだが内容一致，仮定違い，背景説明に近い，講義主張とは別物に分ける。
- 集中講義中に挙げられた参考文献は，登録だけで完了扱いにせず，本文引用，台帳照合，記号確認，未確認点のどれに使ったかを記録する。
- Codex 提案参考文献は，講義中に実際に参照された文献として扱わない。
- LMFDB は具体例，不変量，標準的な記号の確認に限って使う。検索条件，URL，確認した対象，本文での用途を記録し，証明補完や講義由来文献の代替には使わない。

## 用語・記号

- 手書きノートで英語表記の用語は，無理に日本語化しない。
- 日本語用語に英語表記が添えられている場合は，用語直後に括弧付きで英語を置く。
- 未確認の訳語は本文へ直接入れず，`TERM_DICTIONARY.md` の候補欄に残す。
- チルダ系の `\tilde{}`，`\widetilde{}`，`\sim` と，ハット系の `\hat{}`，`\widehat{}` は混同しない。
- 参考文献側の記号へ本文を寄せる必要がある場合は，理由を `WORK_LEDGER.md` に記録する。

## 停止条件

次のいずれかに当たる場合は，本文へ推測で入れず，`WORK_LEDGER.md` に未解決点として残す。

- 手書きノートの該当箇所が判読できない。
- 参考文献の対応箇所は見つかったが，仮定や対象が講義中の主張と一致しない。
- 手書きノートと参考文献の記号が衝突し，採用記法を決められない。
- レポート問題，欄外注，矢印注記の対象範囲を画像上で確定できない。
- 証明を補った方が自然に見えるが，手書きノートに根拠がない。
- ユーザー確認でロック済みの範囲を変更する必要がある。
