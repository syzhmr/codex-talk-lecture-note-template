# 入力資料台帳

このファイルは，手書きノート，板書写真，スライド，配布資料，開催情報を記録する台帳である。
本文整理や参考文献確認では，物理フォルダだけで資料を探さず，まずこの台帳を確認する。
ユーザーが `sources/` 以下に入れた資料は，対象範囲に関係する限り，この台帳へ登録した上で積極的に確認する。

旧版資料，抽出画像，OCR 出力がある場合は，現行作業で使うものをこの表で明示する。
現行版でない資料を新規の本文整理に使わない。
手書きノート，板書写真，スライド，配布資料，録音，文字起こし，参考文献 PDF，抽出画像，作業画像は，プロジェクト同期に必要な資料として原則 Git で追跡する。
GitHub の 100MB 制限，公開範囲，権利上の問題がある資料だけを個別例外として除外し，除外理由，代替ファイル，分割方針，保管場所を注意欄または関連 README に記録する。

## 色・記号の凡例

プロジェクトごとに意味を確認してから埋める。
判読できない場合は推測で本文へ混ぜず，`SOURCE_GAPS.md` に残す。

| 表記 | 意味 | 運用 |
| --- | --- | --- |
| 赤字 | [例: 重要・修正・強調] | 本文へ反映する前に根拠を確認する。 |
| 青字 | [例: 補足・疑問・講義中のコメント] | 講義記録として整理する。証明補完には使わない。 |
| 黄色 | [例: 参考文献候補] | `HANDWRITTEN_REFERENCE_INDEX.md` に記録する。 |
| 矢印 | [例: 対応・引用・主張の接続] | 始点・終点・周辺語句を記録する。 |
| 丸囲み・下線 | [例: 強調・要注意] | 強調対象が主張か文献候補かを分ける。 |
| 斜線・取り消し | [例: 不採用] | 取り消された流れを採用しない。追記側を確認する。 |

## 確認ルール

- 手書きノートを最初に確認する。
- 板書，スライド，配布資料は手書きノートの確認・補強に使う。
- ユーザーが追加した資料は，資料種別，ファイル名，対象範囲，確認状態を登録してから使う。
- 登録済み資料は，対象話題ブロックに関係する限り，本文整理，引用確認，記号確認，資料間照合に使う。
- ページ範囲ごとの一覧画像だけで確認済みにしない。
- 重要な参考文献候補は，該当ページまたは画像を個別に確認する。
- 色付き矢印や引き出し線の先に参考文献候補がある場合は，出典候補として記録する。
- 色付き矢印や引き出し線の先に数学的主張がある場合は，参考文献候補とは別に，講義中の主張候補として記録する。
- 本文へ反映した場合は，反映先のファイル名と位置を記録する。
