# 講演記録作成方針

このファイルは，集中講義で記録された手書きノートを，後から読める自己完結した講義ノートとして TeX 化するための全体方針である。
作業の入口は必ず `WORKFLOW_STEPS.md` とし，プロジェクト固有情報は `PROJECT_PROFILE.md`，資料の所在は `SOURCE_INDEX.md`，日程・講義回・話題ブロックの対応は `SESSION_INDEX.md` を優先する。
単発講演にも転用できるが，運用上の判断は集中講義記録を主対象として行う。

## 基本方針

- 完成版は，手書きノートの PDF 化を主眼にする。
- 手書きノートに書かれている内容は，判読不能箇所を除き，すべて TeX 打ちする。
- 作成手順は，まず手書きノートの数学的主張，証明，具体例，復習，欄外メモをそのまま TeX 打ちし，その後で方針に従って整形する。
- 「記録」とは手書きノートを要約することではなく，手書きノートに含まれる情報を 100\% 以上の情報量で残すことを意味する。
- 完成 PDF は，手書きノートや作業台帳を見なくても本文だけで読める形にする。
- 完成 PDF は，プロジェクトで指定した想定読者に共有しても耐える品質を目標にする。想定読者に専門家が含まれる場合は，専門読者が検証できる講義ノートとして整える。
- 品質目標に教科書並みと指定された場合は，厳密さ，情報量，記号の一貫性，参考文献対応，証明の追跡可能性を重視する。
- 完成 PDF の冒頭には，集中講義名，実施大学，時期，講演者，ノート，目次をこの順に置く。
- テンプレートには具体的な集中講義名，大学名，時期，講演者，ノートを入れず，プロジェクトごとに埋める。
- PDF 冒頭の人物情報は `講演者: 氏名（所属）` と `ノート: 氏名` を標準とし，講演者を根拠なく `講師` と書かない。
- 各日の本文冒頭には日付を表示し，記録としていつの講義内容かが PDF 本文だけで分かるようにする。日付は目次へ入れず，直後の節見出しに付く形で置く。
- AI を利用して作成した場合は，概要に，ノート作成者の手書きノートを主資料にしたこと，AI を補助的に使用したこと，内容の最終確認および誤りの責任がノート作成者にあることを明記する。
- 使用 AI の名称，モデル名，推論設定は，実際に使用した環境に合わせて断定的に書く。テンプレートには特定のモデル名を固定しない。
- 日程，講義回，節題，話題ブロックの順序を保存し，教科書風の章構成へ勝手に組み替えない。
- 節題，小見出し，番号付け，定義・定理・命題などの環境見出しは手書きノートに従う。
- 手書きノートで題名として書かれた語句は，本文側でも対応する見出しとして反映する。
- 手書きノートの見出し階層が明確な場合は，本文側でもその階層を尊重する。
- 定義している概念名，直前の節題，説明用の整理語を，定義・主張などの任意見出しとして補わない。
- 本文の地の文でも，手書きノートにない分類語，説明語，評価語を AI 側で補わない。
- 例や計算が並んでいるだけの箇所に，手書きノートにない `...の例として`，`中心的な条件`，`...を制御する` などの説明表現を足さない。
- 手書きノートにない `まず`，`次に`，`最後に`，`から始まる`，`最初は` などの接続語や導入文を補わない。
- 定理・命題などの環境見出しが人名または人名列の場合は，題名本文ではなく括弧付きの帰属として出す。
- 講義中に記録された流れ，主張，記号，図式，引用先，参考文献対応を重視する。
- 講義中に書かれた曲線図，グラフ，可換図式，矢印図は省略せず，本文の近い位置で TeX 図式または画像として反映する。
- 可換図式が講義記録・手書きノート・板書・配布資料にある場合は，文章説明だけで省略せず，本文へ反映後に対応箇所が残っているか確認する。
- 可換図式や代数的な矢印図は TikZ-cd を最優先にし，曲線図やグラフは TikZ を最優先にして本文へ起こす。
- 可換図式は J. S. Milne, *The tikz package* / *Guide to Commutative Diagrams* (<https://www.jmilne.org/not/Mtikz.pdf>) を組版参照にし，`tikzcd` で読みにくい場合だけ `matrix of math nodes` や直接 `\node` と `\path` による配置へ進む。
- 同型記号は，通常の同型関係には `\cong` を使い，射の矢印の上に同型であることを表示する場合は `\sim` を使う。
- 図式を本文へ入れた後は，PDF または画像化したページで矢印の向き，ラベル位置，対象の重なり，ページ幅からのはみ出しを確認する。
- 生成画像は使用しない。AI 生成画像や画像生成ツールで作った模式図・グラフを，手書きノート，板書写真，参考文献の代替として使わない。
- TikZ または TikZ-cd でどうしても再現が難しい場合だけ，手書きノート，板書写真，参考文献の該当箇所をトリミングして取り込むことを検討する。
- トリミング画像を使う場合は，著作権上の安全性と講義記録の忠実性のため，ユーザー作成ノート由来の画像を最優先にする。参考文献由来のトリミング画像を使う場合は，本文へ取り込む前にユーザーの許可を取る。
- 手書き画像を貼る場合，画像内に残る文章・式・ラベルが直前直後の TeX 本文と接続しているかを確認する。接続しない文字が残る場合は，文字を TeX 化するか，画像を図だけに切る。
- 手書き画像内の文章を読ませる形にしない。画像は図形情報を保つために使い，文章・式・証明・注釈は原則として本文へ起こす。
- 手書きノートに書かれた文章，式，証明，図式，矢印，欄外メモは，判読不能箇所を除き，すべて文章化して本文へ反映する。
- 講義冒頭や途中にある復習，今日の目標，前日からの接続は，講義の構造情報として本文へ反映する。
- 手書きノートに `proof` または `Proof` とある箇所は `証明` として出す。
- 手書きノートに `proof sketch`，`sketch`，または `証明の sketch` とある箇所は `証明の sketch` として出し，別表記へ言い換えない。
- 証明内の主張・補題・claim・lemma は，手書きノートがそのように明示的に切り出している場合に限って，本文でも `主張` や `補題` として切り出す。
- 手書きノートで一続きの証明として書かれている部分を，読みやすさのために講義ノート以上の粒度で新たな主張・補題へ分割しない。
- 用語と表現は手書きノートを優先し，手書きノートで英語表記の語は無理に日本語へ訳さない。
- 手書きノートを最優先資料にする。
- 板書写真，スライド，配布資料は，手書きノートの記録を確認・補強するために使う。
- ユーザーが `sources/` や `references/` に入れた資料は，台帳化した上で，対象範囲に関係する限り積極的に使う。
- 参考文献は，講義中の主張がどの文献のどの定理，命題，節，ページ，URL に対応するかを確認するために使う。
- 手書きノートと配布資料に入っている参考文献を最優先し，講義由来の引用候補として確認する。
- Codex が講義資料の外から提案した参考文献は，講義由来の文献と区別し，引用未特定，記号衝突，標準出典確認，書誌情報確認の補助として使う。
- 一般的に飛ばされている背景や行間を確認する文献は，補足引用に留め，講義記録の主線を置き換えない。
- 証明の不足部分を Codex が補完しない。
- 参考文献を見て，講義中に記録されていない中間補題，証明手順，背景定理を本文へ勝手に追加しない。
- 講演者が述べたと確定できない内容を，講演者の主張として書かない。
- 記号，式，定義の順序は，原則として講義中の記録を基準にする。
- 参考文献側の記法に合わせて講義記録の記法を不用意に置き換えない。
- 記号衝突がある場合は `NOTATION_CONFLICTS.md` に記録する。

## 完成版の性格

完成版は録音の逐語録ではない。
ただし，手書きノートに書かれた内容については，本文へ TeX 打ちすることを基本とする。
講義中に記録されていない内容を滑らかに補って教科書風にすることは目的ではない。

完成版では次を目指す。

- 集中講義の日程，講義回，節題，話題順が追える。
- 手書きノートの記述が，抜粋ではなく本文として残っている。
- 講義中に強調された主張，例，注意，記号が残っている。
- 参考文献の対応箇所が分かる。
- 数学的主張の仮定，結論，記号，証明の依存関係が，専門読者に検証可能な形で書かれている。
- 断片的なメモではなく，講義ノートとして引用・共有できる体裁になっている。
- 未確認箇所や資料間衝突を本文に曖昧に混ぜない。
- 本文だけで読める形にし，証明の不足は作業台帳で管理する。

## 完成 PDF の禁止事項

完成 PDF は，読者に作業過程を読ませるためのファイルではない。
次の表現は完成 PDF 本文に出さない。

- 「手書きノート p.X」「手書きノートでは」「ノートでは」などのページ参照や作業資料名。
- 「記録」「未確認」「再確認が必要」「この版では」といった作業状態の表示。
- 手書きノートにない任意見出しを定義・定理・命題などに付けること。
- 定義している語そのものや節題を，`定義 概念名` のように任意見出しへ重複させること。
- 人名だけの環境見出しを `定理 著者名` のように本文見出しとして出すこと。
- 判読不能，引用未特定，添字未確認を理由にした本文の省略。
- 「流れだけ」「概要だけ」「ページ順の主張だけ」など，全転記でないことを前提にした説明。
- 手書きノートにある証明，証明の sketch，具体例，復習を省略して，結果だけを本文に残すこと。
- 手書きノートで題名・小見出しになっている語句を，本文側で見出しにせず地の文へ吸収すること。
- 可換図式を文章説明だけに置き換えて，本文から図式として消すこと。
- 生成画像を図式，グラフ，曲線図，模式図として本文に入れること。
- ユーザー許可なしに参考文献由来のトリミング画像を本文に入れること。

ページ対応，判読不能，引用未特定，添字未確認，資料間衝突は `SOURCE_GAPS.md`，`CLAIM_REFERENCE_MAP.md`，`SESSION_INDEX.md` に置く。
完成 PDF で必要な場合は，本文として数学的に読める文へ直すか，確定した短い補足引用として入れる。

手書きノートの総ページ数より完成 PDF が短い場合は，未転記が残っている可能性が高いため，完成扱いにしない。
ページ数を水増しするのではなく，各ページに含まれる定義，式，主張，証明の sketch，欄外メモ，図式を本文へ起こしたかで判定する。

## 本文に入れるもの

- 講義中に記録された定義，主張，命題，補題，定理，例，注意。
- 手書きノートに書かれた式，計算，証明，証明の sketch，欄外メモ，矢印，曲線図，グラフ，可換図式。
- 手書きノートの節題，小見出し，番号付け。
- 手書きノートで題名化されている話題名，証明名，復習見出し，番号付き見出し。
- 手書きノートに書かれた定義名，定理名，命題名，補題名，主張名。
- 手書きノートに残っている講義の流れ。
- 板書写真で確認できる式，図式，見出し，記号。
- スライドや配布資料で確認できる正式な主張，図，参考文献。
- 講義中に参照された文献の対応箇所。
- 参考文献で確認した記号差分，仮定差分，定理番号。
- 一般的に飛ばされている背景を確認するための最小限の補足引用。

本文に補足を入れる場合は，講義本文と補足引用を区別する。
補足は，証明や紹介された主張・事実が参考文献のどこにあるかを示すためのものとし，手書きノート本文を置き換えない。

## 本文に入れないもの

- 講義中に記録されていない証明補完。
- 参考文献だけから得た追加の中間補題。
- Codex 提案参考文献だけから得た講義外の補足。
- 補足引用を根拠にした本文の大幅な拡張。
- 講義の流れにない背景説明。
- 手書きノートの節題や用語を，PDF の構成に合わせて言い換えた見出し。
- 手書きノートにない説明用ラベルを，定義・定理・命題などの見出しとして追加したもの。
- 定義・主張の中で本文として自然に述べられる概念名を，任意見出しとして重複させたもの。
- 講演者が述べたと確定できない解釈。
- 仮定や結論の一致を確認していない参考文献由来の主張。
- 判読できない URL，文献名，定理番号を推測で復元したもの。

## 参考文献対応

講義中の主張を本文へ整理する前に，必要に応じて `CLAIM_REFERENCE_MAP.md` を更新する。
対応文献が見つかった場合でも，次を区別する。

- 完全一致
- 記号違いだが内容一致
- 仮定が一部違う
- 結論が類似
- 関連箇所だが対応未確定
- 未発見

本文では，対応が確定したものだけを明確な引用として使う。
未確定のものは本文に混ぜず，台帳に残す。
補足引用は，講義中の主張に直接対応する主引用と区別し，必要最小限の注または短い確認に留める。

## 資料名の扱い

PDF 本文では「手書きノートでは」「スライドでは」「板書では」と作業資料名を不用意に出さない。
完成 PDF では作業資料名を本文の主語にしない。
資料差分や出典の性質を残す必要がある場合は，原則として作業台帳に分離し，本文には確定した数学的内容だけを置く。

作業台帳では，資料名，ページ，画像番号，スライド番号，配布資料の節番号を明確に書く。

## 用語・記号

- 講義中の記法を最優先にする。
- 用語は手書きノートの表現を最優先にする。
- 手書きノートで英語表記の用語は，必要がない限り英語のまま本文に残す。
- 日本語化する場合は，手書きノート内に日本語表記がある場合を優先し，なければ `TERM_DICTIONARY_CANDIDATES.md` に候補として残す。
- 手書きノートの記号が不明確な場合は，板書，スライド，配布資料で確認する。
- 参考文献の記号と講義中の記号が異なる場合は，どちらを採用したかを `NOTATION_CONFLICTS.md` に記録する。
- 用語が未確定または不自然な場合は，本文へ直接入れず `TERM_DICTIONARY_CANDIDATES.md` に回す。

## 査読

通常査読では，手書きノート，板書写真，スライド，配布資料，参考文献を見てよい。
独立査読では，完成版 PDF，引用箇所，参考文献対応，記号衝突を中心に確認する。
独立査読で出た修正文案，判断が分かれる補足，資料間の衝突は `REVISION_CANDIDATES.md` または `SOURCE_GAPS.md` に記録する。

## 報告

作業後は，次を簡潔に報告する。

- 対象日程・講義回・話題ブロック。
- 使用した手書きノート，板書，スライド，配布資料。
- 抽出した参考文献候補。
- 対応付けた講義中の主張と参考文献箇所。
- 未確定の引用，記号衝突，資料間衝突。
- 本文変更とビルド結果。
