# LaTeX ソース可読性方針

このファイルは，LaTeX ソースを人間と AI の両方が読みやすい形に保つための補助方針である。
講演記録の内容，資料対応，参考文献引用，用語に関する判断は，必ず `WORKFLOW_STEPS.md`，`TALK_RECORD_POLICY.md`，`SOURCE_CAPTURE_POLICY.md`，`REFERENCE_CITATION_POLICY.md`，`TERM_DICTIONARY.md` を優先する。

## 優先順位

1. `WORKFLOW_STEPS.md` で作業型を確認する。
2. ユーザーの明示的な指示を最優先する。
3. 講演記録の内容については `TALK_RECORD_POLICY.md`，資料確認については `SOURCE_CAPTURE_POLICY.md`，引用については `REFERENCE_CITATION_POLICY.md` を優先する。
4. 既存のファイル構成，既存マクロ，既存のビルド方法を優先する。
5. この可読性方針は，意味を変えない整形・構造化・命名・差分管理のために使う。

## ファイル構成

- `main_final.tex` は文書全体の構造を管理する。
- `preemble.tex` は既存のプリアンブルとして扱う。ファイル名は現状を尊重し，指示なく変更しない。
- `final/talk01.tex` などの親ファイルは，講演・講義回の見出しと話題ブロックへの `\input` を管理する。
- 各話題ブロックの本文は，`final/talk01/01-overview.tex` のように小さいファイルに置く。
- 完成 PDF 末尾の参考文献欄は `final/references.tex` に置く。

## ソース整形

- 本文は原則として 1 文 1 行にする。
- 段落途中で意味のない改行を大量に入れない。
- 長すぎる行は，文，式，条件，理由などのまとまりで分ける。
- 長い行を分割するときは，非空白文字列が変わらないようにする。
- `\href{...}{...}` や `\url{...}` の長い URL は，リンク先が変わる可能性があるため無理に途中で分割しない。
- 空行は，段落，大きな構造，意味の切れ目を示すために使う。
- 連続する空行は原則 1 行にまとめる。
- 環境，箇条書き，複数行数式は，既存の書き方に合わせて読みやすくインデントする。

## マクロとプリアンブル

- 既存マクロの意味を変更しない。
- 既存マクロと似た役割の新規マクロを不用意に増やさない。
- パッケージ読み込み順，LaTeX エンジン，文書クラスは指示なく変更しない。
- 使われていないパッケージの削除は，コンパイル確認と影響確認ができる場合だけ行う。

## ラベルと参照

- 既存ラベルは，不用意に変更しない。
- 新しいラベルを追加する場合は，節は `sec:`，定義は `dfn:`，定理は `thm:`，補題は `lem:`，主張は `claim:`，命題は `prop:`，系は `cor:` を用いる。
- 話題ブロックには `block:` を用いる。
- 数式ラベルを追加する場合は `eq:` を用いる。
- 意味の薄いラベル名は避ける。
- 再参照しない補助的な記録には不要なラベルを付けない。

## 数式と証明

- 数式の意味，仮定，結論，番号の有無を変えない。
- 番号付き数式を番号なしに変更しない。番号なし数式を番号付きに変更しない。
- 講義中に記録された証明スケッチは整理してよい。
- 講義中に記録されていない証明補完を本文へ追加しない。
- 参考文献に証明がある場合でも，講義記録の不足部分を補う目的で本文へ展開しない。
- 証明補完が必要に見える場合は，`SOURCE_GAPS.md` に記録する。

## 図表

- 図表の内容，順序，位置指定は指示なく変更しない。
- 図表を追加・差し替え・再生成する場合は，講義中に記録された情報として必要かを確認する。
- 板書写真の図式を LaTeX 図式へ起こす場合は，板書の矢印，対象，ラベルを確認し，推測で補わない。

## 引用・参考文献

- 引用キー，引用位置，参考文献情報を推測で変更しない。
- DOI，URL，出版年，著者名などの書誌情報は，信頼できる情報源で確認できる場合だけ補う。
- 本文で引用する前に `CLAIM_REFERENCE_MAP.md` で対応状態を確認する。
- 未確認の引用候補は本文へ出さず，`HANDWRITTEN_REFERENCE_INDEX.md` または `SOURCE_GAPS.md` に残す。
- 完成 PDF 末尾の参考文献一覧は，この記録で実際に使った文献だけを載せる。

## ビルド

- 既存の `latexmkrc` と現在のビルド方法を優先する。
- TeX 本文を変更した場合は，原則として `latexmk -g -dvi -interaction=nonstopmode main_final.tex` でビルドする。
- ビルド成功時に `main_final.pdf` と `main_final.synctex.gz` が生成されることを確認する。
- 生成物や補助ファイルは，指示がない限り編集対象にしない。

## Git

このテンプレートを用いるプロジェクトでは Git 利用を必須とする。
ソース整形や本文整理の前に `git status --short --branch` を確認し，作業後に `git diff` と `git diff --check` を確認する。
内容変更と整形変更は commit を分ける。
手書きノート，板書写真，スライド，配布資料，参考文献 PDF，抽出画像は原則として Git に入れない。

## 報告

作業後は，次を簡潔に報告する。

- 変更したファイル。
- 主な変更内容。
- 実行したビルド，整形，検査コマンド。
- コンパイル結果。
- 未確認引用，記号衝突，資料間衝突。
- 意味を変えた可能性がある箇所があれば，必ず明示する。
